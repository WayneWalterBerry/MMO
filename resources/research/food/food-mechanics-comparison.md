## 15. Lessons by Game Type

### Simulation Games (Dwarf Fortress, CDDA)
- **Strength:** Depth, emergent gameplay, realism
- **Weakness:** Steep learning curve, micromanagement
- **Lesson:** Complexity must be meaningful; every system should create interesting decisions

### MUDs (Achaea, Discworld)
- **Strength:** Non-intrusive, roleplay-friendly, low barrier
- **Weakness:** Minimal mechanical depth
- **Lesson:** Food should enhance experience, not interrupt flow

### Roguelikes (NetHack, Caves of Qud)
- **Strength:** Risk/reward depth, permanent consequences, emergent stories
- **Weakness:** Steep learning, can feel punishing
- **Lesson:** Identification gameplay is engaging; risk creates memorable moments

### Survival Games (Valheim, Don't Starve)
- **Strength:** Strategic buff systems, clear goals, progression
- **Weakness:** Can become repetitive grind
- **Lesson:** Buffs > hunger bars; food as empowerment > food as chore

### Text Adventures (Zork)
- **Strength:** Atmospheric, sensory-rich, puzzle potential
- **Weakness:** Underutilized mechanically
- **Lesson:** Food is narrative gold; use sensory verbs, create memorable interactions

### Board Games (Agricola, Feast for Odin)
- **Strength:** Resource pressure creates strategic tension
- **Weakness:** Abstract (not immersive)
- **Lesson:** Food as economic pressure works if players have meaningful choices

## 17. Recommended Hybrid Model for MMO

**Inspiration Blend:**
- Dwarf Fortress: Emotional system (preferences), cooking as preservation
- NetHack: Risk/reward identification, sensory detection
- Valheim: Buff system (empowerment, not punishment)
- MUDs: Non-intrusive, optional engagement
- Text IF: Sensory richness, puzzle integration

**Core Mechanics:**
1. **Food provides buffs** (Valheim model), not survival pressure
2. **Cooking via mutation** (raw-X.lua → cooked-X.lua using FSM)
3. **Spoilage via time-based FSM** (fresh → spoiling → spoiled)
4. **Sensory identification** (smell safe, taste risky)
5. **Creature interaction** (feed rat, use as bait)
6. **Tool requirements** (fire_source, knife, preservation_tool)
7. **Material integration** (food objects have material properties)

**Phased Rollout:**
1. Basic consumables + buffs (Phase 1)
2. Cooking + mutation (Phase 2)
3. Spoilage + identification (Phase 3)
4. Preservation + containers (Phase 4)
5. Recipes + creature feeding (Phase 5)
