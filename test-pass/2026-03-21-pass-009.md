## Bug Report

### BUG-033: Material "oak" used by objects but missing from registry
- **Severity:** LOW
- **Objects:** nightstand.lua, vanity.lua, wardrobe.lua
- **Input:** N/A (data issue)
- **Output:** `materials.get("oak")` returns nil
- **Expected:** Either add "oak" to registry or change objects to `material = "wood"`
- **Impact:** Any engine code that looks up material properties for these objects will get nil. Currently no gameplay impact because material properties aren't used in FSM threshold checks yet.

### BUG-034: Material "velvet" used by curtains but missing from registry
- **Severity:** LOW
- **Objects:** curtains.lua
- **Input:** N/A (data issue)
- **Output:** `materials.get("velvet")` returns nil
- **Expected:** Either add "velvet" to registry or change curtains to `material = "fabric"`
- **Impact:** Same as BUG-033 — no current gameplay impact.

### BUG-035: GOAP candle relight selects spent match instead of fresh match
- **Severity:** MEDIUM
- **Steps to reproduce:**
  1. Light candle via GOAP (works first time)
  2. Extinguish candle (`blow candle`)
  3. Try to relight: `light candle`
- **Input:** `light candle` (with extinguished candle in hand, matchbox with spent + fresh matches in other hand)
- **Output:**
  ```
  You'll need to prepare first...
  Your hands are full. Drop something first.
  The match is spent. It cannot be relit.
  You have nothing to relight it with.
  ```
- **Expected:** GOAP should skip spent matches and select a fresh (unlit) match from the matchbox.
- **Root cause:** After the first GOAP light chain, the spent match ends up in the matchbox. GOAP's match selection doesn't filter by state — it grabs the first match it finds, which is the spent one.
- **Impact:** Player cannot relight candle via GOAP after first extinguish. Must manually remove spent match from matchbox, then manually light.
- **Workaround:** Manually remove spent match from matchbox (`get spent match` → `drop spent match`), then manual light sequence.
