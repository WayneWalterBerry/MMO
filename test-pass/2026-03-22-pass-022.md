## Detailed Bug Report

### BUG-069: Confusing "sleep until dawn" error after dawn
**Severity:** MINOR (usability)  
**Type:** Error message clarity  

**Input:**
```
> sleep until dawn
```
*When current time is 8:38 AM (already past dawn)*

**Expected:**
```
It's already past dawn.
```
*Or:* "Dawn has already passed. Current time is 8:38 AM."

**Actual:**
```
You can't sleep that long. Try 12 hours or less.
```

**Analysis:**
The error message is technically correct (sleeping until next dawn would be ~22 hours), but confusing to players. The parser should detect when the target time is "until dawn" and the current time is already past dawn, and provide a more contextually appropriate message.

**Recommendation:** Add special case in sleep verb:
```lua
if target_time == "dawn" and current_time > dawn_time then
  return "It's already past dawn. Current time is " .. format_time(current_time) .. "."
end
```

**Impact:** LOW — Only affects players who try to sleep until dawn after dawn has passed. Does not break gameplay.
