## 🔴 BUG-063: CRITICAL — GUID Format Mismatch Breaks All Object Loading

**Severity:** CRITICAL  
**Status:** Game is completely unplayable  
**Discovered:** 2026-03-21 Pass-018  

### Summary
Commit 2799ff6 added braces to 44 object GUIDs but did NOT update room files. The loader performs exact string matching, so all object instantiation fails.

### Technical Details
**Broken Code Path:**
1. `main.lua` line 138: `base_classes[def.guid] = def` → keys with `"{d40b15e6-...}"`
2. Room files reference: `type_id = "d40b15e6-..."` (no braces)
3. `loader/init.lua` line 123: `base_classes[instance.type_id]` → looks for `"d40b15e6-..."` 
4. String comparison: `"{d40b15e6-...}" != "d40b15e6-..."` → NO MATCH
5. Instance resolution fails, object never created

**Scope of Damage:**
- `type_id` with braces: 0 instances
- `type_id` without braces: 79+ instances across 7 room files
- **ALL object instantiation is broken**

### Evidence
**Startup warnings:**
```
Warning: base class not found for guid 'b7c3d1a2-4e5f-4890-ab12-cdef34567890' (instance 'large-crate')
Warning: surface 'sarcophagus-5.inside' not found for instance 'tome'
```

**In-game:**
- Nightstand missing from bedroom
- Cannot obtain matches
- Cannot create light
- Game unplayable from turn 1

### Expected vs Actual
**Expected:** Objects instantiate normally, nightstand present in bedroom  
**Actual:** Objects fail to load, room appears empty

### Fix Options
**Option A (Recommended):** Update all room files to use braced GUIDs
```lua
type_id = "d40b15e6-..." → type_id = "{d40b15e6-...}"
```

**Option B:** Add GUID normalization to loader
```lua
local function normalize_guid(guid)
  return guid:gsub("^{(.-)}", "%1")  -- strip braces
end
```

**Impact:** Until fixed, the entire game is unplayable.
