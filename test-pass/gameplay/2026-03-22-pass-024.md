### Critical Bugs (🔥 P0)

#### **BUG-076: Game hangs on "find something to light"**
**Input:** `find something to light`  
**Output:** [No response — infinite loop or deadlock]  
**Expected:** Should either find matchbox OR respond "I don't understand 'something to light'."  
**Severity:** CRITICAL (P0) — Game crash  
**Impact:** Player must restart game  
**Root Cause:** Parser likely enters infinite loop when processing multi-word abstract target ("something to light").  

---

#### **BUG-077: Game hangs on "search for a match"**
**Input:** `search for a match`  
**Output:** [No response — infinite loop or deadlock]  
**Expected:** Should find matchbox or matches inside matchbox.  
**Severity:** CRITICAL (P0) — Game crash  
**Impact:** Player must restart game  
**Root Cause:** Likely same as BUG-076 — multi-word or ambiguous target parsing.  
**Note:** May be duplicate of BUG-076 (same root cause).

---

#### **BUG-075: `search nightstand` finds nothing despite drawer/matchbox**
**Input:** `search nightstand`  
**Output:** 
```
You begin searching...
You feel the a small nightstand — nothing there.
You finish searching the area. Nothing interesting.
```
**Expected:** Should discover drawer and/or reveal matchbox inside drawer.  
**Actual:** Reports "nothing there" even though drawer exists (with matchbox inside).  
**Severity:** CRITICAL (P0) — Breaks discovery mechanic  
**Impact:** Players using natural "search X" command will miss critical items.  
**Workaround:** Use `examine nightstand` (discovers drawer) → `open drawer` → `feel drawer` (finds matchbox).  
**Note:** This is a REGRESSION — `search around` correctly opens drawers automatically, but `search nightstand` does not.

---

### High Priority Bugs (⚠️ P1)

#### **BUG-074: "look for X" triggers "look" instead of "find X"**
**Input:** `look for the matchbox`  
**Output:** `It is too dark to see anything.`  
**Expected:** Should trigger `find matchbox` behavior (traverse/search).  
**Actual:** Parser interprets as `look` command, ignores "for the matchbox".  
**Severity:** HIGH (P1) — Very common natural phrasing  
**Impact:** "look for X" is natural English — players expect it to search, not just look.  
**Workaround:** Use `find X` or `search for X`.  

---

### Medium Priority Bugs (📌 P2)

#### **BUG-073: "search the room" interprets "the room" as object name**
**Input:** `search the room`  
**Output:** 
```
You begin searching for the room...
[...searches for object named "the room"...]
You finish searching. No the room found.
```
**Expected:** Should work like `search around` (area search).  
**Actual:** Parser interprets "the room" as a target object name.  
**Severity:** MEDIUM (P2) — Common natural phrasing  
**Impact:** "search the room" is very common player input. Should be synonymous with "search around".  
**Workaround:** Use `search around` or `search` without target.
