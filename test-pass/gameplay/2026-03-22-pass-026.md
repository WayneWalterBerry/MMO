## Nightstand Interaction Chain Assessment

### Intended Flow
```
feel around → discover nightstand
feel nightstand → discover drawer handle, top items
open drawer → access inside surface
feel inside drawer → discover matchbox
examine matchbox → discover matches
take match → acquire match
light match → light the match
light candle → light the candle ← BLOCKED (BUG-090)
```

### Actual Results
| Step | Status | Notes |
|------|--------|-------|
| feel around → nightstand | ✅ Works | Clean discovery |
| feel nightstand → drawer | ✅ Works | Good tactile feedback |
| open drawer | ✅ Works | FSM transition correct |
| feel inside drawer → matchbox | ⚠️ Partial | Shows matchbox but also leaks top surface |
| examine matchbox → matches | ✅ Works | Shows all 7 matches |
| take match | ✅ Works | First time only (BUG-091 after spent match exists) |
| light/strike match | ✅ Lights | But dies instantly in same turn |
| **light candle** | **🔴 HANGS** | **Game-breaking. Goal planner stuck.** |

### Verdict: **BLOCKED AT STEP 8**
The nightstand discovery chain is polished and atmospheric through step 7. The final critical step — lighting the candle — hangs the game. This is a **release-blocking** issue.

## Recommendations

### Immediate (P0)
1. **BUG-090:** Fix `light candle` goal planner hang. This is the single most important bug in the game. Consider:
   - Simplify goal plan: if player has match in hand, strike + light in one action
   - If no match in hand but matchbox nearby, auto-take + strike + light
   - Add timeout/safety limit to goal planner (similar to BUG-076/077 fix)

2. **BUG-091:** Fix match pickup priority — fresh matches in matchbox should be preferred over spent matches on floor. Or better: don't let players pick up spent matches at all.

### Short Term (P1)
3. **BUG-092:** Fix status bar match counter to decrement on use.
4. **BUG-089:** Scope `feel inside X` to only show X's interior, not parent surfaces.

### Design Consideration
The match burning out immediately in the same turn as lighting is a design issue. Either:
- **Option A:** Give match a 2-3 turn burn time so player can `light match` then `light candle` manually
- **Option B:** Keep instant burnout but make `light candle` the primary command that auto-handles everything (requires BUG-090 fix)

Option B is preferred as it's more player-friendly, but both approaches need the goal planner to work.
