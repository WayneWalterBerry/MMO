## Bug Reports

### BUG-036: Container can be placed inside itself 🟡 Major

**Steps to reproduce:**
```
> light candle       (GOAP lights room)
> drop spent match
> get matchbox
> put matchbox in matchbox
```

**Expected:** "You can't put something inside itself." or similar rejection.

**Actual:**
```
> put matchbox in matchbox
You put an open matchbox in an open matchbox.
> inventory
  Left hand: a lit tallow candle
  Right hand: (empty)
```

**Impact:** The matchbox vanishes from the player's hand and is logically placed "inside itself." The object is effectively lost — it no longer appears in room contents, player inventory, or anywhere discoverable. This is a containment system bug that could cause item loss with any container.

**Severity:** 🟡 Major — causes item loss. Any container object is vulnerable.

**Root cause (suspected):** The `put X in Y` verb doesn't check if X == Y before performing the containment operation.

## Design Issues (Not Bugs — Feedback for Bob)

### DESIGN-001: "then" not recognized as compound separator

**Input:** `open drawer then get matchbox`
**Output:** `You don't see that here.`

The parser recognizes "and" as a compound separator (`get candle and light it` works beautifully) but "then" is treated as part of the object noun phrase. Players commonly use "then" to chain actions.

**Recommendation:** Add "then" to compound separators alongside "and".

### DESIGN-002: Torch bracket should accept candle placement

**Input:** `put candle in torch bracket` / `put candle on torch bracket`
**Output:** `an iron torch bracket is not a container`

A torch bracket described as "empty" strongly implies the player should be able to place a candle/torch in it. This is a natural affordance that players will try.

**Recommendation:** Add a surface/slot to the torch bracket that accepts candle-type or torch-type objects.

### DESIGN-003: Welcome text hardcoded for `--room` backdoor

**Input:** `lua src/main.lua --no-ui --room cellar`
**Output:** Still shows "THE BEDROOM — A Text Adventure" banner and "You wake with a start" text.

While the `=== DEBUG: Starting in room 'cellar' ===` message is present, the hardcoded intro text is misleading when starting in a non-bedroom room.

**Recommendation:** Either suppress the welcome text in debug/backdoor mode, or make it room-aware. Minor — only affects testers using `--room`.
