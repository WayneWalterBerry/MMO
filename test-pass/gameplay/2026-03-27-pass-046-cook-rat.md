# Pass-046: Cook Dead Rat via Cellar Brazier

**Date:** 2026-03-27
**Tester:** Nelson
**Build:** Lua src/main.lua --headless
**Scope:** Rat combat → corpse pickup → cook mutation via brazier fire source → sensory verification

## Executive Summary

**Total tests:** 8 | **Pass:** 7 | **Warn:** 1 | **Fail:** 0 | **Hang:** 0

The cook-rat pipeline works end-to-end. The player can navigate to the cellar, kill the rat in melee combat, pick up the dead rat, and cook it using the brazier as a fire source. The mutation from `dead-rat` → `cooked-rat-meat` fires correctly, sensory text (`feel`, `smell`) is rich and thematic, and inventory reflects the cooked item. One display bug found: cooked rat meat appears in both the room description AND the player's inventory simultaneously.

## Bug Detail

### BUG-165: Cooked rat meat shows in room description while held in inventory

**Severity:** MEDIUM
**Repro:** `goto cellar` → `attack rat` → `take dead rat` → `cook rat` → `look` then `inventory`
**Observed:** `look` output includes "There is a piece of cooked rat meat here." AND `inventory` shows "Left hand: a piece of cooked rat meat" — the item appears in both places.
**Expected:** If the player is holding the item, it should NOT appear in the room's object list. Room_presence should filter out items in the player's hands.
**Impact:** Confusing for the player — they might think there are two pieces of meat, or try to pick up something they're already holding.
