### Methodology

Two automated test approaches were used:
1. **Pipe-mode test (v1):** Launch game process, pipe stdin/stdout, send input, send `quit`, verify process exits. All 50 passed.
2. **Precise timing test (v2):** Launch game process, measure exact milliseconds from input to first output response. All 50 responded in <2 seconds (threshold: 5 seconds).

**Note on prior interactive false positives:** Earlier manual testing via interactive terminal (PTY) appeared to show hangs on some inputs (e.g., "what do I do?", "what now?"). These were **false positives** caused by the TUI screen renderer — the game redraws the screen using cursor positioning, and some responses overwrite existing screen positions without producing new scrollable content visible to the terminal capture tool. The automated pipe-based tests confirmed the game engine processes these inputs in <1 second.
