## Executive Summary

**Total Tests:** 22 | **Passed:** 17 | **Failed:** 3 | **Hang:** 2 | **Pass Rate:** 77%

Container sensory gating for the **nightstand drawer** works correctly — closed drawer hides contents from both `feel` and `examine`. Opening the drawer reveals the matchbox. The `close` → `examine` → `open` → `examine` cycle gates properly.

**However, the wardrobe is broken.** The wardrobe reports as closed (`"It is already closed"`) but reveals its contents to both `feel wardrobe` and `look inside wardrobe`. This is the primary bug found.

GOAP auto-chain (`light candle` from cold start) works beautifully. Search system works. Critical path is intact.

## Bug Details

### BUG-095: Wardrobe container gating broken (HIGH)
**Repro:**
1. Start game
2. `feel wardrobe` — shows "inside: a moth-eaten wool cloak, a burlap sack"
3. `close wardrobe` → "It is already closed."
4. `look inside wardrobe` (with light) — still shows contents

**Expected:** Closed wardrobe should hide its contents, same as closed drawer hides matchbox.
**Actual:** Wardrobe contents visible regardless of open/closed state.
**Note:** The nightstand drawer gating works perfectly. The wardrobe likely needs the same `is_open_for_sensory` gating applied.

### BUG-096: Gating message references parent instead of target (MEDIUM)
**Repro:**
1. `feel inside drawer` (closed) → "You can't reach inside **a small nightstand**."
2. `feel in drawer` (closed) → "You can't reach in **a small nightstand**."

**Expected:** Message should reference "the drawer" since that's what the player targeted.
**Actual:** Message references the parent container "a small nightstand".

### BUG-097: Closed drawer with light shows description instead of gating message (LOW)
**Repro:**
1. Light candle
2. Close drawer
3. `look inside drawer` → Shows drawer dimensions, no "it's closed" hint

**Expected:** "The drawer is closed." or similar feedback.
**Actual:** Physical description of drawer, but no matchbox contents (gating works functionally).
