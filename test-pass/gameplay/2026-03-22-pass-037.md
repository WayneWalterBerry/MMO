## Bug Details

### BUG-124: `move <object>` interpreted as `go <direction>`

**Severity:** HIGH
**File:** `src/engine/verbs/init.lua`
**Root cause:** Line 2617 defines `handlers["move"]` for spatial object movement. Line 5407 overwrites it: `handlers["move"] = handlers["go"]`. The go-synonym wins because it's defined later.
**Impact:** "move rug", "move bed", "move barrel" all fail. "shift" and "slide" work as workarounds.
**Reproduction:** `move rug` → "You can't go that way."
**Fix suggestion:** Remove the `handlers["move"] = handlers["go"]` synonym, or create a disambiguation handler that checks whether the noun is a direction vs. an object.

### BUG-125: `containers.is_container()` ignores `categories` field

**Severity:** HIGH
**File:** `src/engine/search/containers.lua` (lines 16-19)
**Root cause:** Function only checks `object.is_container == true` or `object.container == true`. Most container objects (14/19) declare container status via `categories = {..., "container"}` without a boolean flag.
**Impact:** Search cannot peek into or report contents of wardrobe, barrel, crates, sarcophagi, offering bowl, vase, rain barrel, well, or any FSM-managed container. This breaks #24 (peek) and #27 (content reporting) for these objects.
**Reproduction:** `find sword` → wardrobe says "nothing there" (expected: list wardrobe contents)
**Fix suggestion:** Add categories check: `or (object.categories and table_contains(object.categories, "container"))`

### BUG-126: Accessible containers don't report contents when target not found

**Severity:** MEDIUM
**File:** `src/engine/search/traverse.lua` (line ~372-416)
**Root cause:** The #27 content-reporting code exists in the inaccessible/peek path (line 358-368) but NOT in the accessible surface path. When a container is open and the target isn't found inside, the step returns with an empty narrative.
**Impact:** Even after opening the wardrobe, searching for an item not inside it doesn't report what IS there.
**Reproduction:** `open wardrobe` → `find sword` → wardrobe says "nothing there" (expected: "Inside you see wool cloak, burlap sack, but no sword")
**Fix suggestion:** Add content reporting after the accessible surface content loop (after line 413), mirroring the peek path logic.
