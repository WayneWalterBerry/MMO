## Hang Pattern Analysis

The 5 hangs share common characteristics:

| Bug | Input | Likely Cause |
|-----|-------|-------------|
| BUG-105 | `what do I do?` | "what do I do" → transforms to something unparseable |
| BUG-106 | `what now?` | "what now" → transforms to bare/empty command |
| BUG-116 | `look around` | "around" treated as target noun rather than room-look modifier |
| BUG-117 | `where is the matchbox?` | "where is" question transform produces unparseable command |
| BUG-118 | `peek behind the curtains` | "peek" unrecognized verb or "behind" causes parser loop |

**Common thread:** All hangs appear to involve the parser entering an infinite loop when it receives a command it can recognize partially but can't fully resolve. The parser likely needs a bailout/timeout mechanism.
