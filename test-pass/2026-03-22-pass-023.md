## BUGS DISCOVERED

### 🐛 BUG-070: Rendering Issue - Command Prompt Scrolls Off Screen
**Severity**: Minor  
**Description**: During long output sequences, the `>` prompt scrolls far up the terminal, making it hard to see where to type next. Output includes excessive blank lines.  
**Repro**: Execute `search around` or `find bed` - output pushes prompt up ~20+ lines  
**Impact**: Player loses orientation. Feels unresponsive even though game is working.  
**Suggested Fix**: Reduce blank line padding between responses. Keep prompt near bottom of visible area.

---

### 🐛 BUG-071: Potential Hang on Rapid `look around` Command
**Severity**: Major  
**Description**: When testing `look around` after previous commands, the game appeared to hang - no output for 20+ seconds. Had to kill session and restart.  
**Repro**: Sequence was: `search around` → `search` → `find nightstand` → `find furniture` → `find something` → `search for nightstand` → `look around` [HANG]  
**Impact**: Game becomes unresponsive. Player thinks it crashed.  
**Suspected Cause**: Output buffer issue? Too many accumulated items in darkness? Rendering loop?  
**Priority**: **CRITICAL** - investigate immediately. Could be edge case or serious blocker.

---

### 🐛 BUG-072: Screen Flicker/Redraw During Progressive Object Discovery
**Severity**: Polish  
**Description**: During `search around`, objects appear incrementally with full screen redraws between each. Creates flicker effect.  
**Observed**: "a large four-poster bed" [redraw] "a large four-poster bed / a small nightstand" [redraw] etc.  
**Impact**: Visually jarring. Feels like loading lag.  
**Suggested Fix**: Batch the full list before displaying, OR use smoother incremental append without full clear.

## GOAP PARSER ASSESSMENT

### ✅ **Core Discovery Chaining: WORKS BEAUTIFULLY**

The GOAP parser successfully chains multi-step discovery:
1. **`search around`** → discovers objects exist
2. **`find nightstand`** → examines object, mentions drawer handle
3. **`open drawer`** → parser remembers context, opens it
4. **`search drawer`** → discovers matchbox inside
5. **`get matchbox`** → takes it
6. **`take match from matchbox`** → nested extraction works
7. **`light match`** → uses held item

**This is exactly what Wayne wanted.**

### ✅ **Minimum Valid Expressions: CONFIRMED**

Players don't need magic words:
- ✅ "open drawer" works (not "open nightstand drawer")
- ✅ "search drawer" works (not "examine inside of drawer")
- ✅ "take match from matchbox" works (not "open matchbox" + "get match")

### ✅ **Natural Language Tolerance: GOOD**

- ✅ Strips prepositions: "search for X" = "find X"
- ✅ Defaults intelligently: "search" = "search around"
- ✅ Context-aware: "open drawer" knows which drawer after finding nightstand

### ⚠️ **Needs Testing: Hidden Object Discovery**

**Critical gap**: Did NOT test whether `find matchbox` works BEFORE opening drawer.

**Expected behavior**: Should FAIL - matchbox is hidden inside closed container.

**Risk if it works**: Breaks discovery logic. Player could skip "open drawer" step.

**Test this immediately before shipping.**
