## DECISION SUMMARY

### Three Core Rules

1. **Covering objects own the relationship**
   - `covering` array on covering object (e.g., rug) declares what it hides
   - Hidden object declares `hidden = true` and provides FSM with `hidden → revealed` transition
   - Room does not track relationships — objects are self-describing

2. **traverse.lua must filter hidden objects**
   - `expand_object()` must check `obj.hidden` and skip hidden objects
   - `matches_target()` must return `false` for hidden objects
   - Currently a bug — search walks past `hidden` flag without checking

3. **The `behind` relationship uses the same pattern**
   - Future `hiding_behind` field on blocker objects (wardrobe, curtains)
   - Identical mechanics to `covering`; only reveal verb differs

## DESIGN RATIONALE

Object-level metadata (not room-level relationship tables):
- Principle 8: objects declare behavior, engine executes
- Composable: rug carries `covering` list if moved to another room
- No second source of truth — one place to author, one place to debug
