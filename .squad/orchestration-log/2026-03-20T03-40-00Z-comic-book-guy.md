## Key Decisions Established

1. **Single-File Architecture** — All parts and parent logic in one Lua file
2. **Part Factory Pattern** — Factory functions instantiate parts as independent objects
3. **FSM State Naming** — `{base_state}_with_PART` and `{base_state}_without_PART`
4. **Verb Dispatch for Parts** — General verbs trigger detachment; parts define verb aliases
5. **Contents Preservation** — Container parts carry contents when detached (by default)
6. **Two-Handed Carry System** — Objects have `hands_required` property (0/1/2 hands)
7. **Reversibility as Design Choice** — Each part's reversibility is a design-time decision
8. **Non-Detachable Parts Valid** — Parts can have `detachable = false` for description-only
