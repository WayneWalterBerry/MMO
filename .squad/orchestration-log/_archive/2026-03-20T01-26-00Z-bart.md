## Implementation Details

### Wearable System Architecture
- **Slot Model:** Object-owned metadata (`wear = { slot, layer }`)
- **Conflict Resolution:** One inner + one outer per slot; unlimited accessories
- **Vision Blocking:** Wear property (`blocks_vision = true`) checked independently
- **Legacy Support:** Objects with `wearable = true` default to `torso/outer`
- **Data Structure:** Flat list (`player.worn`, same pattern as `player.hands`)

### Verbs Implemented
- **WEAR** — Equip item to slot, check conflicts, update worn list
- **REMOVE** — Dequip item from slot, update worn list
- **PUT ON** — NLP preprocessing alias for WEAR
- **TAKE OFF** — NLP preprocessing alias for REMOVE

## Design Rationale
1. **Object Autonomy:** Matches D-14 principle (objects own their state)
2. **Extensibility:** String-based slots enable content authors to create new body locations without engine changes
3. **Simplicity:** Flat worn list sufficient at current scale; indexed maps add complexity without current benefit
4. **Validation:** Content-driven slot names prevent hardcoding; system is maximally flexible
