## Research Summary

**Wasmoon Viability:** HIGHLY VIABLE

**Key Findings:**
1. Wasmoon compiles official Lua 5.4 to WASM — exact version match
2. Engine is pure Lua with only 6 self-contained `require` calls and no C extensions → ~90% unmodified
3. Three browser adaptations needed:
   - `io.popen` for directory listing → replaced by build-time manifest
   - Blocking REPL loop → replaced by event-driven `process_command()` function
   - `print`/`io.write` → overridden to write to DOM

**Performance:**
- Total download: ~168KB gzipped
- Cold start: ~100–200ms
- Per-command latency: <5ms
- Memory: ~5MB
- Smaller and faster than most web pages

**PWA Architecture:**
- No framework needed (vanilla HTML/JS)
- Service worker for offline play
- ~100 lines of JS total

**Prototype Estimate:** 5–7 hours total
- Phase 0 (hello world): 30min
- Phase 1 (one module): 1hr
- Phase 2 (all modules): 1–2hr
- Phase 3 (browser REPL): 2–3hr
- Phase 4 (PWA wrapper): 1hr

**Recommendation:** Proceed with prototype. Create `main_browser.lua` as parallel entry point — don't modify `main.lua`.
