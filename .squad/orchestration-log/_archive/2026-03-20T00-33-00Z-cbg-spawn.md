## Design Decisions Captured
1. **Wear slots defined on object:** e.g., `wear_slot = "head"`, `wear_slot = "torso"`
2. **Slot conflict rules:** Only one item per slot unless layering allowed
3. **Layering system:** Objects define `wear_layer = "inner"` or `wear_layer = "outer"`
4. **Dual-property objects:** Backpacks are wearable + containers; pots can be worn as helmets
5. **Chamber-pot inheritance:** Pots are base class; chamber-pot inherits wearability

## Wearable System Principles
- Objects own their wear metadata (engine just enforces conflicts)
- Extensible without engine changes (new slots invented per-object)
- Gameplay effects encoded in object properties (sack on head = blindness)
- New slots (finger, hands, etc.) don't require engine updates
