## Key Design Decisions Embedded

1. **Pronoun Resolution:** "it"/"this"/"that" resolve to last-examined object (simple, fits terse IF style)
2. **Darkness Verbs:** FEEL/SMELL/TASTE/LISTEN all have dark-aware variations (sensory feedback replaces visual)
3. **Tool Verbs:** WRITE/CUT/SEW/STRIKE/PRICK have tool-present and tool-absent variations
4. **Bare Commands:** Object-requiring verbs (TAKE/OPEN/LIGHT) prompt when called bare
5. **Compound Actions:** STRIKE, SEW, PRICK SELF have two-object or special-case variations
6. **Edge Cases:** Pronouns, ambiguous targets, non-standard phrasings documented for QA

## Design Philosophy

**Darkness is not a wall, it's a different mode of play.** Every sense gives different information about the same object. TASTE is the "learn by dying" sense that teaches caution and consequence.

## Training Pipeline Integration

1. **Bart's scripts** read all ~400 variations
2. Generate embedding vectors for each
3. Group by canonical verb (training label)
4. Train embedding model to cluster variations
5. Produce lookup table: embedding → verb

**QA Phase:** Takes subset of variations, validates embedding matcher returns correct canonical verb.
