### 2026-03-21T12:40Z: User directive
**By:** Wayne Berry (via Copilot)
**What:** Puzzles must be classified into three categories: (1) In the game — implemented and working, (2) Wanted — designed and approved but not yet built, (3) Theorized — conceptualized but not signed off or no place for yet. This allows Bob to design puzzles ahead of time without requiring full implementation. All puzzles must also be rated for hardness/difficulty. Puzzle DESIGN docs go in docs/design/puzzles/. Actual puzzle documentation (per-puzzle specs) stays in docs/puzzles/.
**Why:** User request — enables pipeline of puzzle ideas from concept → design → implementation without bottlenecking on building.
