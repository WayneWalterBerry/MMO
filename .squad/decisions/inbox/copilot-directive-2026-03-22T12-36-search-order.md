### 2026-03-22T12:36: User directive — Search ordering within objects
**By:** Wayne (Effe) Berry (via Copilot)
**What:** When searching an object with surfaces AND embedded containers (like the nightstand):
1. Search surfaces FIRST (top, shelf, etc.)
2. Then search nested containers (drawer, compartments)
3. Objects can have nested objects inside them — surfaces before nesting, always
Example: nightstand → check top surface first (candle holder, bottle) → then open drawer → check drawer contents (matchbox)
This applies recursively — if a container inside an object has its own surfaces, surfaces first, then deeper nesting.
**Why:** User design decision — surface items are more accessible/visible than items buried in containers
