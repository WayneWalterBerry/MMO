### 2026-03-21T17:45: User UX feedback from play testing
**By:** Wayne Berry (via Copilot)
**What:** Four UX improvements from Wayne's play testing:
1. **Room titles should be bold** — when entering a room, the room name needs to stand out visually
2. **Short room descriptions on revisit** — after the first visit, re-entering a room should show a brief description, not the full text. Full description only on `look`. Engine needs to track visited rooms.
3. **Command input styling** — light gray for user input doesn't stand out enough, but bold isn't right either. Need a better visual treatment (maybe a different color like cyan or white vs the gray output text).
4. **Bug report metadata** — `report bug` should include Level name, Room name, last 50 lines displayed (not just 20 commands), and leave a blank section for the user to describe the issue.
**Why:** Play testing feedback — these are polish items that affect the feel of the game.
