## Decisions & Architecture Notes

### 1. File Split Pattern (Bart)
- Actions extracted to separate module to preserve headroom through Phase 4
- Dependencies managed via action_helpers table + M.* wrapper closures
- Test monkey-patching preserved (resolution at call time, not initialization)

### 2. Verb Implementation Pattern (Smithers)
- **Instantiation:** Inline loop with nil-guard on ctx.object_sources (not using shared spawn_objects helper to avoid nil crashes in test mocks)
- **Time advancement:** Uses ctx.time_offset += 5/60 hours + 1 FSM tick (matching rest.lua pattern; no ctx.game:advance_time() API exists)
- **Corpse removal:** Matches both obj.id and obj.guid (handles both real engine IDs and test GUID mocks)

### 3. Object & Creature Metadata (Flanders)
- Spider-meat GUID generated locally (Bart's pre-assignment missing spider-meat entry)
- Venom risk higher for spider-meat (30%) vs cooked-bat-meat (10% food-poisoning) — reflects spider's venomous nature
- Tool resolution dual-pattern (provides_tool + capabilities) until Bart consolidates tool API in Phase 4 cleanup
