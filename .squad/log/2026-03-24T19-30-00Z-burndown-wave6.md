**Deliverables:**

- 24 test cases validating Core Principle 5

**Key Design Decisions:**
- Factory.create_instance(template_id, room_context) creates unique instance
- Automatic GUID assignment (Windows GUID format)
- Template inheritance chain resolution
- State initialization and FSM setup
- Instances are mutable; templates are immutable metadata

**Architecture Notes:**
- Templates stored in `src/meta/objects/` (metadata)
- Instances created at runtime via factory (live objects)
- Registry (src/engine/registry/init.lua) indexes all instances by GUID
- Room instantiation uses factory to populate world
- Factory pattern scales to 100+ objects without collisions

**Implementation Pattern:**
```lua

-- instance now has: guid, template info, state, FSM
```
