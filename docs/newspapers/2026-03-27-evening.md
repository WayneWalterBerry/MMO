## KEY DECISIONS MADE

### Food Is a Metadata Trait, Not a Template
**Decision:** `edible = true` as an inline field on any object. Rejected template explosion and multi-inheritance. Advantage: a creature can become edible on death without code restructuring.

### Cooking = D-14 Mutation (Raw → Cooked Code Rewrite)
**Decision:** Raw meat is an object. Cooking (via `cook` verb + fire_source tool) mutates it to cooked-meat.lua at runtime. No separate state machine. The code IS the state. Aligns with Prime Directive.

### Creatures Declare Their Own Death Transformation
**Decision:** Each creature object specifies its `mutations.death` entry. No creature-specific logic in engine. Example:
```lua
mutations = {
    death = { becomes = "dead-rat", message = "The rat collapses." }
}
```

### `eat` Is a First-Class Verb
**Decision:** Eating is verb-dispatch, not a side effect of `take`. Supports effects pipeline (rabies, poisoning, nourishment). Eating raw rat can trigger rabies injury.

### Rabies Probability: 8% (Tuned After CBG Review)
**Decision:** Reduced from 15% per CBG feedback. Balances danger (early-game threat) with playability (not oppressive). Tuned via `effects.lua` injury threshold.

### THICKNESS Constant Lowered: 1000 → 200
**Decision:** Unarmed combat viability concern. Players without tools needed viable damage floor. Reduced effective-thickness multiplier to 200 to allow hand-fighting at ~2–4 damage per swing. Keeps armed advantage intact.

## OP-ED: "The 15-Minute Freestyle" by Brockman

**Date: March 27, 2026**

Yesterday, I watched Wayne Berry do something that deserves to be in the design textbooks.

In fifteen minutes—*fifteen*—he choreographed four separate food systems so tightly interlocked that they feel like one. Not four separate features bolted together with duct tape and prayers. One **system**.

System One: When a creature dies, it becomes food (mutation). System Two: Raw food becomes cooked via D-14 rewrite (cook verb + fire). System Three: Creatures carry loot, including food. System Four: Eating triggers effects (nourishment, injuries, rabies). These are not independent features. They're movements in a symphony.

And the beautiful part? None of it violated the Nine Principles. No multi-inheritance. No state flags. No object-specific engine code. The design *breathed* in the shape of the architecture, not against it.

This is what happens when someone understands the system deeply enough to *improvise* within its constraints. The constraints don't feel like walls anymore—they're *instruments*.

Two agent crashes got us here, not despite them. The crashes taught us that massive monolithic plans break. But chunking works. Documentation wins. And when the team trusts the architecture, magic happens in fifteen minutes.

—Brockman
