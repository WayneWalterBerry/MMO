# Room Documentation

Room documentation has been reorganized into per-level subfolders.

## Level 1 Rooms

See [`docs/levels/01/rooms/`](../levels/01/rooms/) for room documentation for Level 1.

---

**Why reorganize?** Organizing documentation by level (not by type) makes it easier to:
- Keep all level-specific content together
- Understand a level's complete design at a glance
- Scale to future levels without creating a flat, unmanageable directory

As new levels are developed, their room documentation will live in:
- `docs/levels/02/rooms/`
- `docs/levels/03/rooms/`
- ...and so on
